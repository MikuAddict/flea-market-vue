# 跳蚤市场Vue前端项目

这是一个基于Vue 3 + Element Plus的跳蚤市场前端项目，完全对接真实的8080端口后端API服务。

## 项目特性

- 🛍️ 完整的商品浏览、搜索、购买流程
- 👤 用户注册、登录、个人信息管理
- 📦 商品发布、管理功能
- 💳 订单管理、支付流程
- 📱 响应式设计，支持移动端
- 🎨 现代化的UI界面，基于Element Plus

## 技术栈

- **前端框架**: Vue 3 + Composition API
- **UI组件库**: Element Plus
- **路由管理**: Vue Router 4
- **状态管理**: Pinia
- **构建工具**: Vite
- **HTTP客户端**: 基于OpenAPI Generator生成的API客户端

## 项目结构

```
src/
├── api/           # API接口定义
├── model/         # 数据模型
├── views/         # 页面组件
├── stores/        # 状态管理
├── router/        # 路由配置
└── App.vue        # 根组件
```

## 快速开始

### 环境要求

- Node.js >= 16.0.0
- npm >= 7.0.0

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

项目将在 http://localhost:5173 启动

### 构建生产版本

```bash
npm run build
```

### 预览生产版本

```bash
npm run preview
```

## 后端API配置

项目默认配置为连接到 `http://localhost:8080` 的后端服务。如果需要修改API地址，请修改 `src/ApiClient.js` 文件中的 `basePath` 配置。

## API接口说明

项目使用OpenAPI Generator自动生成的API客户端，所有接口都严格对应后端8080端口提供的服务。主要接口包括：

- 用户相关: 登录、注册、用户信息
- 商品相关: 商品列表、详情、发布、删除
- 订单相关: 创建订单、支付、订单管理
- 分类相关: 分类列表
- 评价相关: 商品评价

## 注意事项

1. 确保后端服务在8080端口正常运行
2. 所有数据交互都使用真实API，无模拟数据
3. 需要先登录才能使用发布商品、购买等功能
4. 图片上传功能需要后端支持文件上传接口

## 开发说明

- 项目使用Vue 3的Composition API编写
- 遵循Vue官方最佳实践
- 使用ES6+语法
- 组件化开发，提高代码复用性
- 响应式设计，适配不同屏幕尺寸

## 部署说明

1. 构建生产版本: `npm run build`
2. 将 `dist` 目录部署到Web服务器
3. 配置服务器支持HTML5 History模式
4. 确保后端API服务可访问
